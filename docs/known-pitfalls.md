
이 문서는 AI가 이전 작업에서 했던 재발 가능성 있는 실수와 주의사항을 기록한다.

기록 원칙:
- 반복 가능성이 있는 실수만 기록한다
- 정책, 구조, 일관성 관련 실수를 우선 기록한다
- 단순 오타, 사소한 실수는 기록하지 않는다

## 2026-04-18

### 1. 전역 도메인 문서만 보고 구현 기준을 고정
- 문제: 도메인 정책이 `docs/domain/*.md`로 분리됐는데 `docs/domain-rules.md`만 읽고 구현하거나 수정 범위를 판단함
- 원칙: `docs/domain-rules.md`는 인덱스로 보고, 실제 정책 판단은 해당 도메인 문서를 기준으로 한다

### 2. 사용자 확인 없이 비단순 작업을 바로 실행
- 문제: 단순 CRUD가 아닌 작업인데 계획 확인 없이 구현을 바로 진행하거나, 프론트까지 함께 수정함
- 원칙: 비단순 작업은 먼저 계획을 작성해 사용자 확인을 받고, 프론트는 명시 요청이 있을 때만 수정한다
